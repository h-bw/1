# PR 标题

补充接口性能测试脚本、测试结果与论文 Word 版说明

# PR 说明

## 变更内容

本次提交围绕论文第 `4.3 系统测试` 中缺失的性能测试部分进行了补充，新增了接口压测脚本、测试结果文件以及可直接插入论文正文的 Word 文档，便于后续在论文中补充“商品查询接口、订单提交接口、推荐接口”的并发测试结果。

## 具体改动

- 新增 `scripts/run_interface_benchmark.py`
  - 用于对商品查询接口、订单提交接口和推荐接口进行并发测试
  - 支持 `10 / 30 / 50` 并发档位
  - 输出平均响应时间、95%响应时间、错误率等核心指标

- 新增 `logs/interface_benchmark_results.json`
  - 保存本次实际运行得到的性能测试原始结果
  - 可作为论文表格与结果分析的依据

- 新增 `scripts/generate_performance_test_word.py`
  - 用于将测试结果自动整理成 Word 文档
  - 便于后续重复生成或更新测试报告

- 新增 `4.3性能测试结果-Word版.docx`
  - 包含可直接插入论文 `4.3` 节的性能测试表格与分析文字

## 测试结果摘要

- 商品查询接口在 `10 / 30 / 50` 并发下错误率均为 `0`
- 推荐接口在 `10 / 30 / 50` 并发下错误率均为 `0`
- 订单提交接口在并发测试中出现较高错误率
  - 进一步排查发现，当前订单号生成方式采用“秒级时间戳 + 用户ID”
  - 在高并发条件下容易产生重复订单号并触发主键冲突
  - 该结果可直接作为论文中“系统局限性与后续优化方向”的依据

## 说明

本次提交主要补充论文性能测试支撑材料，不涉及推荐算法主流程改造。后续如需继续优化订单提交接口，可进一步考虑：

- 使用毫秒级时间戳
- 引入雪花算法
- 使用 UUID 或数据库序列方案生成订单号
