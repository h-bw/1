# 协同过滤改造说明

## 修改背景

原推荐模块主要依据历史订单统计、热门商品及规则加权进行推荐，能够满足基础推荐需求，但尚未形成标准的协同过滤推荐流程。为使推荐模块的实现与论文中“协同过滤推荐”的表述保持一致，现对推荐逻辑进行规范化改造。

## 修改内容

1. 基于订单数据构建用户-商品交互矩阵。
2. 使用余弦相似度计算用户之间的相似度。
3. 根据相似用户的历史行为对候选商品进行加权评分。
4. 对无历史行为或候选商品不足的场景增加热门商品兜底推荐。
5. 补充推荐模块单元测试，验证推荐结果与冷启动场景。

## 当前实现说明

当前实现受现有订单明细表结构限制，订单明细中未直接保存 `product_id`，因此推荐过程中主要依据商品名称构建商品交互信息。该方案可以支撑当前系统中的协同过滤推荐流程，但后续若为订单明细补充 `product_id` 字段，可进一步提升推荐计算的规范性、稳定性与可扩展性。

## 用途说明

本文件用于说明推荐模块从基础规则推荐调整为协同过滤推荐的原因、范围与实现方式，便于项目维护、论文撰写及答辩说明时查阅.
